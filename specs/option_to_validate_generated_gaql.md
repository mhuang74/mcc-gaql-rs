# Plan: Add `--validate` Option to Generate Command

## Context

After GAQL is generated by `mcc-gaql-gen generate`, users currently have no way to verify the query is valid against the actual Google Ads API until they run it with `mcc-gaql`. This feature adds a `--validate` flag that uses Google Ads API's `validate_only=true` parameter to check query validity without executing it.

**Key finding**: Google Ads API validation requires a real account + credentials + customer_id. There is no public API for GAQL validation without authentication.

## Implementation Approach

Reuse `mcc-gaql` crate as a library dependency. It already exports `googleads`, `config` modules with OAuth2 handling, API access, and config resolution. Add a new `validate_gaql_query()` function to `mcc-gaql/src/googleads.rs` that uses `SearchGoogleAdsRequest` with `validate_only: true`.

## Files to Modify

### 1. `crates/mcc-gaql-gen/Cargo.toml`
Add `mcc-gaql` as workspace dependency:
```toml
mcc-gaql = { workspace = true }
```

### 2. `Cargo.toml` (workspace root)
Add mcc-gaql to workspace dependencies:
```toml
mcc-gaql = { path = "crates/mcc-gaql" }
```

### 3. `crates/mcc-gaql/src/googleads.rs`
Add new function:
```rust
/// Validate a GAQL query against Google Ads API without executing it
pub async fn validate_gaql_query(
    api_access: &GoogleAdsAPIAccess,
    customer_id: &str,
    query: &str,
) -> Result<(), anyhow::Error>
```
- Uses `SearchGoogleAdsRequest` (non-streaming) with `validate_only: true`
- Returns `Ok(())` if valid, `Err` with API error message if invalid

### 4. `crates/mcc-gaql-gen/src/main.rs`
- Add CLI args to `Commands::Generate`:
  ```rust
  #[arg(long)]
  validate: bool,

  #[arg(long)]
  profile: Option<String>,
  ```
- Update `cmd_generate()` to:
  1. Always print generated query to stdout first
  2. When `--validate` is set:
     - Auto-detect profile: if `--profile` not specified, use `mcc_gaql::config::list_profiles()` to find profiles; if exactly one, use it; otherwise error with list of available profiles
     - Load config via `mcc_gaql::config::load(profile)`
     - Check if token cache exists; if not, fail with message to run `mcc-gaql --setup` first (no OAuth flow)
     - Call `mcc_gaql::googleads::get_api_access()` to get authenticated API access
     - Call new `mcc_gaql::googleads::validate_gaql_query()` function
     - Print validation result to stderr (PASSED/FAILED)

## Key Implementation Details

1. **Reuse existing code**: `mcc_gaql::googleads::get_api_access()` handles OAuth2, token caching, dev token
2. **Reuse config**: `mcc_gaql::config::load()` and `mcc_gaql::config::list_profiles()`
3. **Profile auto-detection**: Use last/only profile from config.toml if `--profile` not specified
4. **No OAuth flow**: If token cache missing, fail with instructions to run `mcc-gaql --setup`
5. **API Call**: `SearchGoogleAdsRequest` with `validate_only: true` (non-streaming)
6. **Output**: Always print query to stdout; validation result to stderr
7. **Exit codes**: 0 = valid, 1 = invalid query, 2 = error (auth/config)

## Verification

1. `cargo check --workspace` - ensure no circular dependencies
2. Test without `--validate` (should work as before)
3. Test with `--validate --profile <PROFILE>` - valid query
4. Test with `--validate` on deliberately invalid query (should fail)
5. Test `--validate` without credentials (should show helpful error)

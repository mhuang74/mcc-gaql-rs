# Metadata Pipeline Design Specification

**Version:** 1.0
**Date:** 2026-03-11
**Status:** Implemented
**Branch:** `claude/google-ads-metadata-plan-MY2YN`

---

## Overview

This document describes the design of the Google Ads metadata pipeline added to `mcc-gaql-rs`. The pipeline enriches the existing field metadata cache with human-readable descriptions generated by an LLM, enabling higher-quality semantic search and more accurate GAQL query generation from natural language.

The pipeline consists of three stages:

```
Fields Service API
      │
      ▼
┌─────────────────────┐
│  field_metadata.rs  │  Stage 0: Structural harvest
│  FieldMetadataCache │  (fields, types, flags, enum values)
└──────────┬──────────┘
           │
           ▼
┌─────────────────────┐
│ metadata_scraper.rs │  Stage 1: Web scraping
│  ScrapedDocs        │  (plain-text descriptions, enum docs)
└──────────┬──────────┘
           │
           ▼
┌──────────────────────┐
│ metadata_enricher.rs │  Stage 2: LLM synthesis
│  MetadataEnricher    │  (contextual descriptions, usage notes)
└──────────┬───────────┘
           │
           ▼
  field_metadata_enriched.json
  (replaces field_metadata.json)
           │
           ▼
  LanceDB vector store rebuild
  (via lancedb_utils / prompt2gaql)
```

## Module Descriptions

### `src/field_metadata.rs` — Structural Metadata Cache

Manages the local cache of Google Ads field schema obtained from the Fields Service API.

**Key types:**

```rust
pub struct FieldMetadata {
    pub name: String,           // e.g. "campaign.name"
    pub category: String,       // RESOURCE | ATTRIBUTE | METRIC | SEGMENT
    pub data_type: String,      // STRING | INT64 | DOUBLE | ENUM | DATE | ...
    pub selectable: bool,
    pub filterable: bool,
    pub sortable: bool,
    pub metrics_compatible: bool,
    pub selectable_with: Vec<String>,
    pub enum_values: Vec<String>,
    pub attribute_resources: Vec<String>,
    // Populated by metadata_enricher:
    pub description: Option<String>,
    pub usage_notes: Option<String>,
}

pub struct ResourceMetadata {
    pub name: String,
    pub selectable_with: Vec<String>,
    pub key_attributes: Vec<String>,
    pub key_metrics: Vec<String>,
    pub field_count: usize,
    pub description: Option<String>,
}

pub struct FieldMetadataCache {
    pub fields: HashMap<String, FieldMetadata>,
    pub resource_metadata: Option<HashMap<String, ResourceMetadata>>,
    pub api_version: String,
    pub fetched_at: DateTime<Utc>,
}
```

**Key methods:**

| Method | Purpose |
|--------|---------|
| `fetch_from_api(ctx)` | Query all fields from Fields Service gRPC API |
| `load_or_fetch(ctx, path, ttl)` | Load from disk if fresh, else fetch |
| `save_to_disk(path)` | Persist cache as JSON |
| `get_resource_fields(resource)` | List all fields for a resource |
| `get_resources()` | List all top-level resource names |
| `export_summary()` | Human-readable schema summary |
| `show_resources()` | Resource hierarchy table |
| `enriched_field_count()` | Count fields with LLM descriptions |

**Cache location:** `~/.config/mcc-gaql/field_metadata.json`
**Default TTL:** 7 days (configurable via `field_metadata_ttl_days`)

### `src/metadata_scraper.rs` — Web Scraping Stage

Scrapes the Google Ads API reference documentation to extract plain-text field descriptions and enum value definitions.

**Target URL pattern:**
```
https://developers.google.com/google-ads/api/fields/{version}/{resource}
```

**Key types:**

```rust
pub struct ScrapedFieldDoc {
    pub description: String,       // Plain-text, stripped of HTML
    pub enum_values: Vec<String>,  // UPPER_SNAKE_CASE values found in the page
}

pub struct ScrapedDocs {
    pub scraped_at: DateTime<Utc>,
    pub api_version: String,
    pub docs: HashMap<String, ScrapedFieldDoc>,  // field_name → doc
    pub resources_scraped: usize,
    pub resources_skipped: usize,
}
```

**Key methods:**

| Method | Purpose |
|--------|---------|
| `load_or_scrape(resources, version, path, ttl, delay)` | Load cache or scrape from web |
| `scrape_all(resources, version, delay)` | Scrape all resources with rate limiting |
| `scrape_all_with_base_url(...)` | Same but with custom base URL (for testing) |
| `parse_field_docs(resource, html)` | Extract field docs from HTML |
| `get_description(field)` | Look up scraped description |
| `get_enum_values(field)` | Look up scraped enum values |

**Design decisions:**

- **Rate limiting:** 500 ms delay between resource pages (configurable) to respect the docs server
- **Graceful degradation:** Pages that fail or appear JS-rendered return empty results without aborting
- **HTML parsing:** Simple line-by-line extraction keyed on `<h2>/<h3>/<h4>` with `id=` attributes; no full DOM parser needed given the consistent structure of the Google Ads docs
- **Enum extraction:** `UPPER_SNAKE_CASE` token detection in `<td>` and `<code>` tags
- **Skip list:** `metrics`, `segments`, and `accessible_bidding_strategy` are skipped (no dedicated reference pages)
- **Cache location:** `~/.cache/mcc-gaql/scraped_docs.json`
- **Cache TTL:** 30 days

### `src/metadata_enricher.rs` — LLM Enrichment Stage

Uses an LLM to synthesize contextual, search-optimized descriptions for every field, combining structural metadata from the Fields Service with scraped documentation.

**Key type:**

```rust
pub struct MetadataEnricher {
    llm_config: LlmConfig,
    batch_size: usize,  // default 15 fields per LLM call
}
```

**Key methods:**

| Method | Purpose |
|--------|---------|
| `enrich(cache, scraped)` | Enrich all fields in-place |
| `enrich_batch(resource, fields, scraped)` | One LLM call for a batch of fields |
| `enrich_resource(name, rm, scraped)` | Generate resource-level description |
| `parse_enrichment_response(response)` | Parse JSON from LLM |

**Top-level pipeline function:**

```rust
pub async fn run_enrichment_pipeline(
    cache: &mut FieldMetadataCache,
    llm_config: &LlmConfig,
    scrape_cache_path: &Path,
    scrape_ttl_days: i64,
    scrape_delay_ms: u64,
) -> Result<()>
```

**LLM prompt design:**

The system prompt instructs the LLM to produce JSON with two keys per field:
- `description`: 1–2 sentences covering what the field represents and when to use it, phrased to match natural user queries ("show campaign names", "filter by status")
- `usage_notes`: brief notes on filtering, sorting, or common GAQL patterns (optional)

Each batch prompt includes:
1. Field name, category, data type, and capability flags
2. Enum values from the Fields Service (ground truth)
3. Scraped plain-text description (when available)
4. Additional enum context from scraping (when available)

**Batch size:** 15 fields per LLM call balances token budget and cross-field context.

**Error handling:** If an LLM call fails for a batch, the batch is skipped with a warning and enrichment continues. Fields retain their prior (possibly empty) descriptions.

**JSON fence stripping:** The LLM sometimes wraps output in ` ```json ``` ` fences; `strip_json_fences()` removes these before parsing.

## CLI Integration

The pipeline is exposed via new flags in `--help`:

| Flag | Description |
|------|-------------|
| `--refresh-field-cache` | Fetch structural metadata from Fields Service and save to disk |
| `--show-fields <resource>` | Print all fields for a resource (e.g. `campaign`) |
| `--show-resources` | Print resource hierarchy with field counts and key attributes |
| `--export-field-metadata` | Print a full schema summary to stdout |
| `--refresh-metadata` | Run the full pipeline: harvest → scrape → LLM enrich → rebuild vector store |

### `--refresh-metadata` pipeline sequence

```
Stage 0: Fetch structural metadata from Fields Service gRPC API
Stage 1: Load scraped docs from cache or scrape Google Ads API reference
Stage 2: Run LLM enrichment (batched, per-resource)
         → Save enriched cache to field_metadata_enriched.json
         → Overwrite field_metadata.json (so --natural-language benefits immediately)
         → Clear LanceDB vector cache (forces rebuild with richer embeddings on next run)
```

**Required environment variables for `--refresh-metadata`:**

```
MCC_GAQL_LLM_API_KEY   or   OPENROUTER_API_KEY
MCC_GAQL_LLM_BASE_URL
MCC_GAQL_LLM_MODEL
```

## Integration with Natural Language Queries

When `--natural-language` is used, `prompt2gaql::convert_to_gaql_enhanced()` is called if a field metadata cache is present. The enriched descriptions (stored in `FieldMetadata.description` and `.usage_notes`) are included in the text used to build embeddings for the LanceDB vector store, giving the semantic search more signal about each field's purpose and usage patterns.

**Fallback:** If the cache is absent or cannot be loaded, the system falls back to the basic natural language path (cookbook-only RAG), with a warning to run `--refresh-field-cache`.

## Feature Flag

All three modules are gated behind the `llm` feature flag:

```toml
[features]
default = ["llm"]
llm = ["dep:rig-core", "dep:rig-fastembed", "dep:rig-lancedb", "dep:lancedb", ...]
```

Build without LLM features (avoids the `lance` crate compilation issue on Rust 1.94+):

```bash
cargo build --no-default-features
cargo test --no-default-features
```

## Testing

| Test file | Coverage |
|-----------|---------|
| `src/metadata_scraper.rs` (unit tests) | HTML parsing, enum extraction, field ID extraction, description lookup |
| `tests/metadata_scraper_tests.rs` | Integration tests with a mock HTTP server (no network dependency) |
| `tests/metadata_scraper_live_tests.rs` | Live tests against the real Google Ads docs (ignored by default, run with `--ignored`) |

Run integration tests:

```bash
# Unit tests only
cargo test --no-default-features --lib

# Integration tests with mock server
cargo test --no-default-features

# Live tests (hits real Google Ads docs)
cargo test --no-default-features --test metadata_scraper_live_tests -- --ignored --nocapture
```
